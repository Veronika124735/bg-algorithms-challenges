# Предизвикателство: Намиране на липсващата буква — рефакториране

## Инструкции

В предишния раздел създадохме функция наречена `findMissingLetter`, която приема масив от последователни (нарастващи) букви като вход и връща липсващата буква в масива.

Искам да направим същото, използвайки функции от висок ред за масиви. Възможно е да се направи с `map`, `filter`, `reduce` или `forEach`.

### Сигнатура на функцията

```js
/**
 * Returns the missing letter in an array of consecutive letters.
 * @param {string[]} arr - An array of consecutive letters.
 * @returns {string} - The missing letter.
 */
function findMissingLetter(arr: string[]): string;
```

### Примери

```js
findMissingLetter(['a', 'b', 'c', 'd', 'f']); // => "e"
findMissingLetter(['O', 'Q', 'R', 'S']); // => "P"
findMissingLetter(['t', 'u', 'v', 'w', 'x', 'z']); // => "y"
```

### Ограничения

- Входният масив винаги ще съдържа поне две букви
- Входният масив ще съдържа букви само в един регистър (малки или главни)
- Използвайте само функции от висок ред за масиви. Не се допускат for цикли.

### Подсказки

- Можете да използвате `map`, за да получите масив от unicode номерата (използвайки charCodeAt) на буквите във входния масив, и след това да използвате `find`, за да намерите липсващия код на символа.
- Можете също да използвате `filter`, както и `reduce`

### Тестови случаи

```js
test('Find Missing Letter', () => {
  expect(findMissingLetter(['a', 'b', 'c', 'e'])).toBe('d');
  expect(findMissingLetter(['X', 'Z'])).toBe('Y');
  expect(findMissingLetter(['m', 'n', 'o', 'q', 'r'])).toBe('p');
  expect(findMissingLetter(['F', 'G', 'H', 'J'])).toBe('I');
});
```
