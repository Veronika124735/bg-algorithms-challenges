# Логаритмична времева сложност `O(log n)`

Логаритмичната времева сложност е когато времето за изпълнение нараства логаритмично с размера на входа. С увеличаването на размера на входа, времето за изпълнение на алгоритъма се увеличава, но с много по-бавни темпове в сравнение с линейната или квадратичната времева сложност. При алгоритъм с логаритмично време, броят на операциите се мащабира логаритмично с размера на входа.

Логаритмичната времева сложност се означава с Big O нотация като O(log n) или "O от log n", където n представя размера на входа.

## Пример: Функция с логаритмична времева сложност

Разгледайте функция наречена `findPower`, която изчислява степента на число чрез рекурсия. Функцията приема два параметъра:

- `base` - основното число.
- `exponent` - степента, на която се повдига основата.

Ще използваме рекурсия в този пример, която научихме в предишната секция.

```js
function findPower(base, exponent) {
  if (exponent === 0) {
    return 1;
  }

  if (exponent % 2 === 0) {
    const halfPower = findPower(base, exponent / 2);
    return halfPower * halfPower;
  } else {
    const halfPower = findPower(base, (exponent - 1) / 2);
    return base * halfPower * halfPower;
  }
}
```

Във функцията `findPower` използваме рекурсия за ефективно изчисляване на степента на число. Функцията използва свойството, че всяко число, повдигнато на четна степен, може да се раздели на две половини, и резултатът е квадратът на числото, повдигнато на половината от степента. За нечетни степени, функцията намалява степента с 1 и повтаря процеса.

## Ефективност и най-добри случаи за използване

Логаритмичната времева сложност е изключително ефективна и желана за големи набори от данни или проблеми, които могат да бъдат разделени на по-малки подпроблеми. Тя често се среща в алгоритми, които използват стратегии "разделяй и владей" или двоично търсене, за което ще говорим скоро, когато стигнем до структури от данни и алгоритми.

В примера с `findPower`, алгоритъмът ефективно изчислява степента на число, като намалява броя необходими умножения, което го прави идеален за изчисления с големи степени.

## Тестване на времето за изпълнение

Тестването на времето за изпълнение на алгоритми с логаритмично време може да се направи с по-големи размери на входа и наблюдаване на относително бавния растеж на времето за изпълнение в сравнение с линейни или квадратични алгоритми. Имайте предвид обаче, че алгоритмите с логаритмично време вече са изключително ефективни и разликата във времето за изпълнение за умерено големи размери на входа може да не е значителна.

Опитайте да стартирате този код:

```js
console.time('Find Power 1');
findPower(2, 100);
console.timeEnd('Find Power 1');

console.time('Find Power 2');
findPower(2, 1000000000);
console.timeEnd('Find Power 2');
```

Резултатът, който получих, е:

```sh
Find Power 1: 0.048ms
Find Power 2: 0.005ms
```

Времето за изпълнение на `findPower` със степен 1 милиард е малко по-бързо от времето за степен 100. Това е защото алгоритъмът вече е изключително ефективен и разликата във времето за изпълнение не е значителна. Помнете, има много други фактори, които могат да повлияят на времето за изпълнение на алгоритъм, като хардуерната и софтуерната среда.
