# Квадратична времева сложност `O(n^2)`

Квадратичната времева сложност е когато времето за изпълнение се мащабира квадратично с входа. С увеличаването на размера на входа, времето за изпълнение на алгоритъма също се увеличава по квадратичен начин (т.е. времето за изпълнение е пропорционално на квадрата на размера на входа). Ако размерът на входа е 1, времето за изпълнение е 1 стъпка. Ако размерът на входа е 10, времето за изпълнение е 100 стъпки. Ако размерът на входа е 100, времето за изпълнение е 10 000 стъпки. И така нататък.

Това поведение се означава с Big O нотация като O(n^2) или "O от n на квадрат", където n представя размера на входа.

Нека разгледаме пример за функция с квадратично време O(n^2).

```js
function sumArray(arr) {
  let sum = 0;
  let sum2 = 0;
  for (let i = 0; i < arr.length; i++) {
    sum += arr[i];
    for (let j = 0; j < arr.length; j++) {
      sum2 += arr[j];
    }
  }
  return sum + sum2;
}
```

В дадения пример имаме функция наречена `sumArray`, която изчислява сумата на всички елементи в масив. Тази имплементация обаче показва квадратична времева сложност (O(n^2)) поради наличието на вложени цикли.

Външният цикъл итерира веднъж през всеки елемент от входния масив, докато вътрешният цикъл итерира отново през масива за всяка итерация на външния цикъл. В резултат, за всеки елемент, срещнат във външния цикъл, вътрешният цикъл итерира през целия масив.

Това води до значително увеличение на броя извършвани операции с нарастването на размера на входа. За масив с дължина n, общият брой операции става приблизително пропорционален на n * n, което води до квадратична зависимост между размера на входа и времето за изпълнение.

Поради вложените цикли, времето, необходимо за изпълнението на функцията sumArray, нараства експоненциално с размера на входа. С увеличаването на масива, времето за изпълнение нараства много по-бързо, което я прави неефективна за по-големи входове.

Така че като цяло, квадратичната времева сложност не е много ефективна. Има обаче случаи, в които тя е най-доброто или единственото решение.

Можете да тествате времето за изпълнение на вашата машина, като стартирате следния код:

```js
function sumArray(arr) {
  let sum = 0;
  let sum2 = 0;
  for (let i = 0; i < arr.length; i++) {
    sum += arr[i];
    for (let j = 0; j < arr.length; j++) {
      sum2 += arr[j];
    }
  }
  return sum + sum2;
}

const arr1 = [1, 2, 3, 4, 5];
console.time('Sum Array 1');
sumArray(arr1);
console.timeEnd('Sum Array 1');

const arr2 = Array.from({ length: 10000000 }, (_, index) => index + 1);

console.time('Sum Array 2');
sumArray(arr2);
console.timeEnd('Sum Array 2');
```

Може да се наложи да щракнете с десен бутон в `Output` във VS Code и да изберете "Stop Code Run", защото ще отнеме много време да се изпълни `Sum Array 2`.
