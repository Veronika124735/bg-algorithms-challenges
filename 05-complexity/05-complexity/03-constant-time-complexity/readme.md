# Константна времева сложност `O(1)`

Установихме, че `константната` времева сложност е когато времето за изпълнение е винаги едно и също, независимо колко голям е входът. `Линейната` времева сложност е когато времето за изпълнение се мащабира линейно с входа. За да опишем това с Big O нотация, използваме `O(1)` (произнася се "O от 1" или "Big O от 1").

Нека разгледаме пример за функция с константно време O(1).

```js
function accessElement(arr, index) {
  return arr[index];
}
```

Тази функция приема масив и индекс и връща елемента на този индекс. Независимо колко голям е масивът, времето за изпълнение ще бъде винаги едно и също, защото извършва само една операция. Можем да опишем тази функция като `O(1)` с Big O нотация.

Можем да тестваме това във VS Code, като добавим следното в .js файл:

```js
function accessElement(arr, index) {
  return arr[index];
}

const arr1 = [1, 2, 3, 4, 5];
console.time('Access Element 1');
console.log(accessElement(arr1, 1));
console.timeEnd('Access Element 1');

const arr2 = Array.from({ length: 10000 }, (_, index) => index + 1);

console.time('Access Element 2');
console.log(accessElement(arr2, 1));
console.timeEnd('Access Element 2');
```

С функциите `console.time` и `console.timeEnd` можем да видим колко време отнема изпълнението на кода между тях.

`Access Element 1` подава масив с 5 елемента, а `Access Element 2` подава масив с 10 000 елемента.

Стартирайте кода, като щракнете с десен бутон и изберете `Run Code` или като го стартирате в терминала с `node <filename>.js`.

Всички ще получите различен резултат. Както казах в предишния урок, има толкова много фактори, включително вашия компютър, операционна система и какви други програми имате стартирани. Също така, начинът, по който компютърът ви разпределя паметта, може да повлияе на времето за изпълнение. Единствената истина е, че ако увеличите размера на масива, времето за изпълнение ще остане в същия общ диапазон.

Променете дължината на масива в `Access Element 2` на 100 000 и стартирайте кода отново.

```js
const arr2 = Array.from({ length: 100000 }, (_, index) => index + 1);
```

Трябва да видите, че времето за изпълнение все още е в същия общ диапазон.
