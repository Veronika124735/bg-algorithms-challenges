# Какво е пространствена сложност?

Пространствената сложност се отнася до количеството памет или пространство за съхранение, което алгоритъм или функция консумира според размера на входа. Тя измерва как използването на памет на алгоритъма се мащабира с размера на входните данни. Целта е да се оптимизира използването на памет и да се минимизира допълнителната памет, необходима по време на изпълнението на алгоритъм или програма.

Има много прилики между времевата и пространствената сложност. Можем да използваме същата Big O нотация за описване на пространствената сложност. Имаме константна, линейна, квадратична, логаритмична, експоненциална и факториелна пространствена сложност.

В много случаи пространствената сложност на функцията ще бъде същата като времевата сложност. Ако функция отнема `O(n)` време, тя също ще заема `O(n)` пространство. Ако функция отнема `O(n^2)` време, тя също ще заема `O(n^2)` пространство. Това е защото количеството памет, използвано от функцията, често е пряко свързано с броя операции, които извършва. Това обаче не винаги е така. Понякога функцията ще заема повече памет отколкото време, или обратното.

## Правила за изчисляване на пространствената сложност

1. **Променливите и структурите от данни заемат пространство:** Когато декларирате променливи или използвате структури от данни (масиви, обекти и т.н.) във вашия код, те заемат памет според тяхния размер и тип данни. Пространствената сложност се увеличава с броя и размера на променливите и структурите от данни, използвани във вашата програма.

2. **Извикванията на функции заемат пространство:** Всеки път, когато извикате функция, програмата трябва да задели памет за стека на извикванията, който включва аргументи на функцията, локални променливи, адреси за връщане и друга служебна информация. Всяко извикване на функция добавя към пространствената сложност и ако имате вложени извиквания на функции или рекурсивни функции, това може да доведе до значително увеличение на използването на пространство.

3. **Пространствената сложност на функция е сумата от пространствените сложности на всички променливи, структури от данни и извиквания на функции вътре във функцията:** Това означава, че за да изчислите пространствената сложност на функция, трябва да вземете предвид всички променливи и структури от данни, използвани вътре във функцията, и техните изисквания за пространство. Допълнително, ако функцията прави рекурсивни или вложени извиквания на функции, трябва да отчетете и пространството, използвано от тези извиквания.

## Примери за пространствена сложност

Нека разгледаме няколко примера:

## Константно пространство `O(1)`

```js
function add(num1, num2) {
  return num1 + num2;
}
```

Пространствената сложност на тази функция е `константна`. Функцията не създава никакви допълнителни структури от данни или променливи, които зависят от размера на входа. Тя просто връща резултата от събирането, така че количеството памет, използвано от функцията, не се променя с размера на входа.

Можем да опишем тази функция като `O(1)` с Big O нотация.

## Линейно пространство `O(n)`

```js
function createArray(num) {
  const arr = [];

  for (let i = 0; i < num; i++) {
    arr.push(i);
  }

  return arr;
}
```

Пространствената сложност на функцията createArray(num) е `линейна`. Това е защото функцията създава масив и добавя елементи в него чрез for цикъл. С увеличаването на входния параметър `num`, размерът на масива също се увеличава пропорционално. Следователно, пространствената сложност на функцията нараства линейно с входа.

Можем да опишем тази функция като `O(n)` с Big O нотация.

## Квадратично пространство `O(n^2)`

```js
function createMatrix(num) {
  const matrix = [];

  for (let i = 0; i < num; i++) {
    matrix[i] = [];

    for (let j = 0; j < num; j++) {
      matrix[i][j] = i + j;
    }
  }

  return matrix;
}
```

Пространствената сложност на функцията createMatrix(num) е `квадратична`. Това е защото функцията създава двумерен масив и го запълва със стойности чрез вложени for цикли. С увеличаването на входния параметър `num`, размерът на матрицата също се увеличава пропорционално. Следователно, пространствената сложност на функцията нараства квадратично с входа.

Можем да опишем тази функция като `O(n^2)` с Big O нотация.

## Логаритмично пространство `O(log n)`

```js
function findPower(base, exponent) {
  if (exponent === 0) {
    return 1;
  }

  if (exponent % 2 === 0) {
    const halfPower = findPower(base, exponent / 2);
    return halfPower * halfPower;
  } else {
    const halfPower = findPower(base, (exponent - 1) / 2);
    return base * halfPower * halfPower;
  }
}
```

Пространствената сложност на функцията `findPower(base, exponent)` е `логаритмична`. Това е защото функцията използва рекурсия за изчисляване на степента на число. С увеличаването на входния параметър `exponent`, броят на рекурсивните извиквания се увеличава логаритмично. Следователно, пространствената сложност на функцията нараства логаритмично с входа.

Можем да опишем тази функция като `O(log n)` с Big O нотация.

В тези примери времевата и пространствената сложност са еднакви. Това обаче не винаги е така. Нека разгледаме пример, където пространствената сложност е различна от времевата сложност.

## Константно време `O(1)` и Линейно пространство `O(n)`

```js
function findSum(arr) {
  let sum = 0;

  for (let i = 0; i < arr.length; i++) {
    sum += arr[i];
  }

  return sum;
}
```

Времевата сложност на функцията `findSum(arr)` е `линейна` или `O(n)`, където `n` е дължината на входния масив `arr`. Това е защото функцията използва цикъл, за да итерира през всеки елемент от масива и да извърши операция с константно време (добавяне на елемента към сумата). Броят на итерациите е пряко пропорционален на размера на масива.

Пространствената сложност на функцията е `O(1)`, което е `константно`. Това е защото функцията използва само една променлива sum за съхраняване на резултата. Пространството, използвано от променливата, не зависи от размера на входния масив, така че остава константно независимо от размера на входа.
